# Live Lesson Mode — Design

**Date:** 2026-07-23
**Status:** approved for planning
**Owner decision log:** см. «Решения» ниже — все развилки утверждены владельцем в брейншторме.

## 1. Что строим

Режим ведения занятия для преподавателя (аналог classroomscreen + живой контроль группы):
преподаватель ведёт урок с группой, демонстрирует материалы/доску/задания («сцена»),
видит в реальном времени прогресс и сигналы учеников, помогает точечно. Ученики видят
сцену на своих устройствах; для очного класса есть чистое окно под проектор.

## 2. Решения (утверждены)

| Развилка | Решение |
|---|---|
| Модель «экрана класса» | **C**: трансляция сцены на устройства учеников + отдельное чистое окно для проектора |
| Запуск урока | Из расписания/журнала (кнопка у слота) **и** вручную (выбрал группу → начал) |
| Привязка ученика к сцене | **Переключаемая**: тумблер strict (принудительное следование) / free (может отойти, препод видит где) |
| Просмотр черновиков | Снапшоты раз в 5–10 сек с dirty-check; только во время активного урока |
| Транспорт realtime | **SSE + Redis pub/sub** (push сервер→клиент) + REST (действия). Минимальная задержка без стоимости WebSocket |
| Peek («глаз») | Ученику **не** показывать, что препод смотрит его черновик |
| Компоновка экрана препода | **A**: сцена ~70% + постоянная правая панель группы + левая рейка сцен |
| Виджеты classroomscreen (таймер, случайный ученик…) | **Отложено**, проверить после v1 |
| Сценарий урока (заготовленные «сцены» заранее) | Не нужен: режим урока — свободный интерфейс |
| Встроенный Jitsi | **Отложено из v1** (бесплатный tier не гарантирует модераторство; JaaS — платный, требует апрува трат) |
| Совместное рисование / live-курсоры / посимвольный ввод | Вне v1; путь апгрейда — точечный WS-канал рядом с SSE |

## 3. Scope

**In (v1):** доска Excalidraw, показ материалов урока, аннотации поверх материалов,
разбор задания на сцене, показ решения ученика (с анонимизацией), живой ростер группы,
живая сетка прогресса по заданию, просмотр черновика ученика, «выдать задание сейчас»,
подсказка ученику, сигналы (рука/«непонятно»/«готово») и быстрые опросы, посещаемость +
журнал, итоги урока, окно проектора.

**Out (v1):** виджеты classroomscreen, Jitsi-встройка, совместная доска, глобальный
автосейв черновиков вне урока (таблица позволит включить позже), звонковая механика.

## 4. Ключевые понятия

**Сцена** — что преподаватель показывает классу:

| type | payload | Класс видит |
|---|---|---|
| `blank` | — | заставка |
| `board` | `board_id` | доска (read-only) |
| `material` | `lesson_id`, якорь блока, `annotation_board_id?` | теория/видео через content-renderer, поверх — слой аннотаций |
| `task` | `exercise_id` | упражнение открывается интерактивно (плеер V2), ученики решают |
| `solution` | `exercise_id`, снапшот работы, `anonymous: bool` | разбор работы ученика |

Аннотации = Excalidraw-слой с прозрачным фоном (kind=`annotation`, привязан к материалу).
Ограничение v1: для file-upload-заданий в `solution` показываются метаданные, разбор
файла препод ведёт через окно проектора.

**Follow mode** — `strict`: смена сцены принудительно переключает экран ученика;
`free`: ученик может отойти (навигация по LMS), кнопка «вернуться к уроку», препод
видит текущее местоположение каждого.

## 5. Архитектура данных

### Postgres (одна additive Alembic-миграция)

- `live_lessons`: `id`, `org_id`, `group_id → student_groups`, `course_id?`,
  `teacher_id`, `class_session_id?` (журнал), `status` (`active`/`ended`),
  `follow_mode`, `current_scene JSONB` (реплика для восстановления после Redis-рестарта),
  `started_at`, `ended_at`, `summary JSONB` (итоги).
  **Partial unique index `(group_id) WHERE status='active'`** — один активный урок на группу.
- `lesson_boards`: `id`, `live_lesson_id`, `kind` (`board`/`annotation`),
  `scene JSONB` (Excalidraw), `version int`, `material_ref?`, `updated_at`.
  Персист дебаунсом ~3 сек, не на каждый штрих.
- `exercise_drafts`: `exercise_id`, `student_id` (unique пара), `answers JSONB`,
  `source_code?`, `updated_at`. Не привязана к уроку (задел под глобальный автосейв),
  но v1-клиент пишет только при активном уроке.

### Redis (живое состояние, TTL, умирает с уроком)

- `lesson:{id}:scene` — текущая сцена (зеркало `current_scene`).
- `lesson:{id}:presence:{student_id}` — `last_seen`, `current_view`, `current_task`; TTL 15 сек.
- `lesson:{id}:signals`, `lesson:{id}:poll:{pid}` — сигналы, опрос+голоса.
- `student:{id}:active_lesson` — ставится heartbeat'ом (крюк для submit-событий).
- `student:{id}:lesson_invite` — ставится при старте урока каждому участнику (вход, O(1)); TTL 4 ч + явное удаление при end.
- Pub/sub канал `lesson:{id}` — шина SSE между воркерами uvicorn.

### Переиспользуем без изменений

`StudentGroup` (состав класса), `ExerciseSubmission` + плеер упражнений V2 (24 типа),
`ClassSession`/журнал (посещаемость, тема), notifications (подсказки, приглашение в урок),
Excalidraw 0.18 (уже в deps), content-renderer (TipTap-теория, видео).

## 6. API — `/api/v1/live-lessons` (модуль `backend/app/live_lessons/`)

### Преподаватель

| Метод | Путь | Описание |
|---|---|---|
| POST | `/live-lessons` | старт `{group_id, course_id?, class_session_id?}`; при существующем активном сначала проверка протухшести (heartbeat препода >10 мин → авто-энд старого и создание нового), иначе 409 с id живого |
| POST | `/{id}/end` | завершить: summary, посещаемость в журнал, очистка Redis |
| PATCH | `/{id}/scene` | смена сцены; для `solution` сервер сам кладёт снапшот работы в payload (кап 64 КБ) |
| PATCH | `/{id}/settings` | `{follow_mode}` |
| POST | `/{id}/boards` | создать доску `{kind, material_ref?}` |
| PATCH | `/{id}/boards/{bid}` | дельты `{updated: [...], deleted: [ids], version}`; last-write-wins, клиент при разрыве нумерации перезапрашивает полную сцену |
| GET | `/{id}/roster` | стартовое состояние панели (участники, presence, сигналы) |
| GET | `/{id}/progress?exercise_id=` | сетка: сабмиты + свежесть черновиков |
| GET | `/exercises/{eid}/drafts/{student_id}` | **живёт в модуле exercises**, не live_lessons. ACL: роль teacher/admin той же org (как просмотр сабмитов) — черновик не привязан к уроку. `If-None-Match`/updated_at → 304. Поллинг препода 3–5 сек, пока открыт просмотр. Peek-состояния на сервере нет |
| POST | `/{id}/messages` | подсказка `{student_id, text}` → SSE-toast + notification |
| POST | `/{id}/polls`, `/{id}/polls/{pid}/close` | опросы |

### Ученик

| Метод | Путь | Описание |
|---|---|---|
| POST | `/{id}/heartbeat` | раз в 5 сек `{current_view, exercise_id?}`; препод тоже шлёт (авто-завершение) |
| POST/DELETE | `/{id}/signals` | ✋ / «непонятно» / «готово» / снять |
| POST | `/{id}/polls/{pid}/vote` | голос |
| POST | `/exercises/{eid}/draft` | **модуль exercises.** Снапшот черновика (5–10 сек, dirty-check). Гейт на сервере: нет Redis-ключа `student:{id}:active_lesson` → 204 no-op без записи в БД. Поэтому эндпоинт вне rate-limit безопасен |
| GET | `/live-lessons/active` | O(1)-чтение `lesson_invite`-ключа; зовётся один раз при загрузке страницы (баннер «урок идёт»), НЕ в цикле |
| GET | `/live-lessons` | список уроков своих групп (прошедшие занятия — блок у ученика на курсе/дашборде) |

### Общее

- `GET /{id}` — полное состояние: урок, сцена, follow_mode, `my_signal`, `active_poll`
  (вход и реконнект).
- `GET /{id}/boards/{bid}` — полная сцена + version.
- `GET /{id}/events` — SSE-стрим.

**Права:** все эндпоинты org-scoped; препод = `teacher_id` урока или admin; ученик =
член группы. **Rate limits:** heartbeat/draft/board-PATCH — вне slowapi; signals —
под лимитом (анти-спам).

## 7. SSE-протокол

Один стрим на участника. Redis-конверт `{audience: all|teacher|student:{id}, event, data}`,
сервер фильтрует по роли. Keepalive-комментарий каждые 15 сек. Заголовок
`X-Accel-Buffering: no` (nginx-конфиг не трогаем; проверить только read-timeout).
Реконнект = EventSource auto-reconnect → `GET /{id}` + полная доска; Last-Event-ID
НЕ используем (нет серверного буфера). Кап соединений на урок = участники + запас;
переполнение очереди соединения → разрыв, клиент переподключается на свежее состояние.
Авторизация — существующие httpOnly-куки (same-origin через nginx).

| Событие | Кому | Данные |
|---|---|---|
| `scene_changed` | всем | сцена |
| `board_delta` | всем | `{bid, updated, deleted, version}` |
| `settings_changed` | всем | follow_mode |
| `poll_started` / `poll_closed` | всем | опрос / результаты |
| `lesson_ended` | всем | — |
| `message` | адресно ученику | текст подсказки |
| `presence` | преподу | батч; шлётся при смене view/задания/онлайн-статуса (вкл. отвал по TTL), тики `last_seen` НЕ считаются изменением |
| `signal` | преподу | `{student_id, type, on}` |
| `submission` | преподу | `{student_id, exercise_id, passed}` |
| `poll_progress` | преподу | счётчики |

**Крюк в exercises.submit:** одна проверка Redis-ключа `student:{id}:active_lesson` →
publish `submission` в канал урока. Других изменений в exercises нет.

## 8. Протокол доски (самое нагруженное место)

- Пишет доску только преподаватель (проектор и ученики — read-only).
- Клиент шлёт **только изменённые элементы** (diff по Excalidraw onChange), троттл 500 мс,
  кап payload; полную сцену не гоняем.
- `version` инкрементится на каждый PATCH; получатель при пропуске version →
  `GET /boards/{bid}` (полная сцена) и продолжает с дельт.
- Persist в Postgres дебаунсом ~3 сек (отдельно от broadcast).
- **Изображения на доске в v1 не поддерживаются**: Excalidraw хранит их бинарно в
  `files`, одна вставка ломает кап дельт. Кап это и enforce'ит. Files-механизм —
  следующая итерация.

## 9. UI

### Преподаватель — `(admin)/admin/live/[lessonId]`

Точки входа: журнал (кнопка у сегодняшнего слота), страница группы.
Компоновка A: левая рейка сцен · сцена ~70% · правая панель.

- Сцена: доска (полный Excalidraw) / материал (дерево курса → content-renderer,
  «рисовать поверх») / задание (пикер + «выдать группе») / разбор (ученик/сабмит,
  тумблер «анонимно»).
- Правая панель, вкладки: **Группа** (ростер: presence, где сейчас, сигналы; клик →
  drawer: черновик live, сабмиты, «отправить подсказку»), **Задание** (живая сетка:
  решил/пытается/не открыл, попытки, время), **Опрос**, **Журнал** (посещаемость
  автоотметкой из presence + ручная правка, тема, заметки).
- Топ-бар: таймер, онлайн-счётчик, тумблер strict/free, «Проектор»
  (window.open → `/admin/live/[id]/screen`, голая read-only сцена со своим SSE),
  «Завершить».

### Ученик — `(dashboard)/lesson/[lessonId]`

Сцена на весь экран + нижняя панель: ✋ / «непонятно» / «готово», имя сцены.
Задание = плеер V2 inline, черновик автосейвится. Опрос — модалка. Подсказка — toast
+ колокольчик. Strict: без навигации внутри страницы; уход со страницы не блокируем —
presence покажет «не в уроке». Free: глобальный хук (только у участников активного
урока) продолжает heartbeat с текущим маршрутом; кнопка «вернуться к уроку».

### Вход ученика (без нового поллинга)

1. Старт урока → Redis `lesson_invite` каждому + notification (колокольчик уже
   поллится раз в 30 сек).
2. `GET /live-lessons/active` при загрузке страницы → баннер «Урок идёт → войти».
3. Очно препод говорит «заходим» — ученик открывает страницу, слой 2 срабатывает сразу.

## 10. Интеграции

- **Журнал:** старт → привязка/создание ClassSession; конец → посещаемость из presence
  (порог «был ≥5 мин», препод правит), тема/заметки.
- **Итоги и доступ после урока:** `summary` (посещаемость, опросы, показанные сцены,
  выданные задания). Доски и summary хранятся бессрочно. `GET /{id}` работает после
  завершения для всех участников: страница урока переходит в **read-only ревью** —
  ученик видит доски (вкл. аннотации), показанные материалы/задания, результаты
  опросов и свои сабмиты; препод — то же + полную сетку группы. Список прошедших
  занятий — `GET /live-lessons` (блок на курсе/дашборде ученика).
- **Доска → материал курса:** кнопка препода «прикрепить доску к уроку курса» —
  сцена доски сохраняется как обычный контент-блок урока (виден в плеере курса всем,
  включая отсутствовавших).
- **Домашка:** deep-link на существующее создание assignment с предзаполненной группой.

## 11. Отказоустойчивость и защита слабого прода (CX22)

- Снапшоты черновиков: только в активном уроке, dirty-check, троттл.
- Дельты доски: только диф, троттл 500 мс, кап размера.
- Presence: батчи преподу только при изменениях.
- Peek: обычный условный GET (304), нагрузка — 1 запрос/3–5 сек от одного препода.
- Redis-рестарт: сцена из `live_lessons.current_scene`, presence восстановится за 5 сек.
- Препод пропал: heartbeat протух >10 мин → lazy авто-завершение при следующем
  обращении любого клиента (UPDATE WHERE status='active' — гонко-безопасно).
- Фейл POST черновика — молча повтор следующим тиком.

## 12. Тесты

- **pytest (реальный Postgres):** старт/энд, уникальность активного урока, права
  (роль/org), SSE-стрим через httpx, дельты+version, draft upsert+304, submit-крюк,
  авто-завершение.
- **Vitest:** `useLessonChannel` (реконнект, применение дельт), редьюсер сцены.
- **Playwright e2e:** препод стартует → ученик входит → сцена долетает → задание
  выдано → сабмит в сетке → завершение.
- Все новые .tsx — `useTranslation()` (i18n-линт), токены Lively, без белого на жёлтом.
