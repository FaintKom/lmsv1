03 · архитектура

Lesson vs Exercise — путающая двойная иерархия

Что есть. 6 типов урока-контейнера. Внутри interactive — 16 типов упражнений. quiz и code_challenge живут на обоих уровнях.

Проблема. Учитель в course builder видит quiz в двух местах и не понимает разницы.

РекомендацияОставить как есть, но переименовать в UI: quiz (урок) → «Тест», quiz (упражнение) →  «Быстрый вопрос». На уровне данных рефактор рискован — 30+ курсов и история попыток.
P1 · UI only≈ 1 день
B · GAPS

Чего не хватает

3 недостающих типа
04 · gap

Нет «опросов / surveys»

Что не так. Нет типа для несценочных форм: рефлексия после модуля, NPS курса, placement до старта, сбор тем для следующих уроков.

Как сейчас. Учителя костылят это через quiz без баллов или через file_upload — и теряют агрегацию ответов по группе.

ПланНовый тип survey = quiz без grading + дашборд-агрегатор для учителя (распределения по вопросам, фильтры по группам). Технически — 80% переиспользует quiz.
P1≈ 1–2 спринта
05 · gap

Нет «hotspot / image-click»

Что не так. Нет типа «ткни на правильную часть картинки». Без него анатомия, география, схемы делаются текстом или 3D-сценой — overkill.

Зачем. Самый дешёвый из недостающих типов и при этом покрывает 3+ предметных области.

ПланImage + N bbox / точек с tolerance-радиусом. Переиспользует механику drag-pin из math_interactive. Учительский UI — клик по картинке для расстановки точек.
P0≈ 1 неделя
06 · gap

Нет «audio recording»

Что не так. Из 5 языковых упражнений ни одно не проверяет произношение. Без записи речи GrassLMS не закрывает B2B-сегмент языковых школ — там это обязательно.

MVP — закрывает 70%audio_upload: запись через MediaRecorder API → файл → ручная проверка учителем. Добавляется как вариант file_upload.
Полная версияWhisper / Azure Speech: STT + scoring произношения, авто-фидбек. Требует API-бюджета (~$0.006/мин Whisper).
P0 (MVP)P2 (full)3 дня → 2 спринта

07 · UX

Quiz UI — самый бедный из 16

Что не так. QuizExercise — текст + radio, и всё. Нет markdown в вопросе, нет картинок в опциях, нет explain-карточки после ответа. При этом quiz — самый используемый тип; апгрейд UI поднимет качество всех существующих курсов сразу.

Roadmap по приоритетам → P0 markdown в теле вопроса (TipTap уже есть, надо подключить) — 2 дня → P0 картинки в опциях (математика, биология, языки) — 3 дня → P1 explain-карточка «почему» после ответа — 1 неделя → P2 randomize order, partial credit для multi-select
P0 × 2P1P0 ≈ 1 спринт
08 · UX

Fullscreen — белый список из 3-х

Что не так. FULLSCREEN_TYPES = robot_2d, math_interactive, world_3d — хардкод. Web_editor, code_challenge и reading тоже выиграют от fullscreen, особенно на 13'' лэптопах.

ПланСделать fullscreen доступным для всех типов по умолчанию — тоггл у студента. Белый список убрать, прятать только для очевидно коротких: true_false, conjugation row.
P1≈ полдня
09 · UX

Attempts — дефолт 100 попыток

Что не так. Реально это «unlimited». Никакой прессуры «думай, перед тем как ответить» — студенты тыкают ответы подбором.

Рекомендация · дефолты по типу → quiz / true_false / matching: 3 → fill_blanks / categorize: 5 → sentence_builder / translation / conjugation: 10 (язык — ок tries) → code_challenge / web_editor: ∞ (отладка — часть процесса) → robot_2d / math_interactive / world_3d: ∞
P0≈ 1 день
