UX audit — exercise mechanics, batch 1

A code-level review of the six core V2 mechanics (quiz, true/false, matching, ordering, fill-blanks, categorize) plus the shared LessonShell, focused on three goals: kid-friendly clarity (ages 7–14), rich interaction feedback, and real adaptiveness. Every finding links to a fix you can try live in the playground.

Severity scale

BUG

Broken behaviour or a contradiction with the design's own rules. Fix first.

A11Y

Keyboard / screen-reader / WCAG gap. Blocks some children entirely.

UX

Works, but confuses or frustrates a 7–14-year-old. High polish value.

ADAPT

Breaks or degrades at narrow widths / touch. The "must not break on phone" rule.

Each row notes whether the fix is already SHIPPED IN PLAYGROUND or PROPOSED for the production codebase.

1 · LessonShell lesson-shell.tsx · .lf-* / .gp-*

The shared Duolingo-style frame is a strong foundation — progress, hearts, streak, bottom sheet, signature button physics. The issues are in the details that children rely on most: status legibility and the feedback moment.

IDTypeFinding → recommendation
SH-01BUGMobile feedback styles never applyThe @media (max-width:560px) block targets .lf-fb-msg / .lf-fb-explain, but the rendered classes are .lf-fb-text / .lf-fb-sub. Phone users get desktop-size sheet typography in a cramped sheet. SHIPPED IN PLAYGROUND
SH-02ADAPTAdaptive rules key off the viewport, but exercises live in a paneIn the LMS, exercises render beside a sidebar — a 700px pane on a 1280px screen gets desktop styles it can't fit. Converting the .lf-*/.gp-* media rules to container queries (container-type:inline-size on .lf-shell) makes every mechanic adapt to the space it actually has. SHIPPED IN PLAYGROUND
SH-03ADAPTFeedback sheet overflows at 320px.lf-fb-row is a no-wrap flex row: icon + message + reveal + CTA compete for ~296px. Let the row wrap and give the CTA flex-basis:100% below 560px — thumb-friendly full-width button, message gets the full line. SHIPPED IN PLAYGROUND
SH-04A11YResult is never announcedThe sheet swaps in visually but has no role="status"/aria-live — a screen-reader child hears nothing after Check. One attribute fixes it. SHIPPED IN PLAYGROUND
SH-05UXHearts are a number, not heartsmaxHearts is accepted and discarded (void maxHearts); the pill shows "3" → "2". Pre-readers and early readers parse filled vs empty heart icons instantly, and the loss animation lands on the specific heart that broke. SHIPPED IN PLAYGROUND
SH-06UXDisabled Check is a dead endA grey button gives zero guidance about why it's grey. Children tap it anyway — answer with a wiggle + a one-line hint ("Pick an answer first"). Turns frustration into instruction. SHIPPED IN PLAYGROUND
SH-07UXDisabled Skip button when there is nothing to skipQuiz renders Skip with no handler — a permanently dead ghost button. Hide it instead. SHIPPED IN PLAYGROUND
SH-08UXConfetti dies mid-airgp-fall translates a fixed 560px; on tall panes the celebration fizzles at half height. Use container-relative units (110cqh). SHIPPED IN PLAYGROUND
SH-09UXNo reward grammar on success beyond confetti.fb-xp-flyer and flyXP() exist but nothing in the shell mounts an #xp-anchor. Wiring the +XP flight into the success sheet completes the invite→…→reward loop the design system itself documents. PROPOSED

2 · Quiz quiz-v2.tsx

IDTypeFinding → recommendation
QZ-01UXWrong picks are forgotten between retriesAfter "Try again" all tiles reset — a child can re-tap the same wrong answer and lose another heart for nothing. Keep eliminated options visibly struck out and unclickable across retries; this also teaches elimination strategy. SHIPPED IN PLAYGROUND
QZ-02A11YNo keyboard shortcutsTiles are buttons (good), but power-kids and motor-impaired users benefit from 1–9 to select. Small index hints on each tile advertise it. SHIPPED IN PLAYGROUND
QZ-03UXSelection ✓ looks like "correct" ✓The tile dot shows a checkmark the moment you select — same glyph the correct-reveal uses. A child can read mere selection as confirmation. Selection = filled dot; ✓ appears only on grading. SHIPPED IN PLAYGROUND
QZ-04UXEmpty config renders a bare string"No questions configured" in 13px grey is developer UX, not child UX. Friendly illustrated empty state + "ask your teacher" copy. SHIPPED IN PLAYGROUND
QZ-05ADAPTMono font for option textOptions render in --font-mono; long sentences in mono wrap poorly at 320px and read "techy" for young readers. Reserve mono for math/code content (the MaybeMath path), sans for prose options. SHIPPED IN PLAYGROUND

3 · True / False true-false-v2.tsx

IDTypeFinding → recommendation
TF-01BUGThe answer leaks on the first wrong attemptThe tile state logic marks v === correctAnswer as correct whenever feedback is shown — including the "1 try left" sheet. The child sees the green tile, taps "Try again", and copies it. QuizV2 explicitly guards against this ("never leak the answer while retries remain"); TrueFalse contradicts its own family rule. Reveal only when the task ends. SHIPPED IN PLAYGROUND
TF-02UX2 attempts on a binary choice = guaranteed passEven with TF-01 fixed, a retry on true/false is just "pick the other one". Default to 1 attempt and spend the saved moment on the explain text — the learning payload of this mechanic. (Playground: set Hearts to 1 in Tweaks to feel it.) PROPOSED
TF-03UXNetwork failure reads as "you're wrong"A failed onGrade shows the coral ✕ sheet ("Couldn't submit"). The child did nothing wrong — and on flaky school Wi-Fi this is common. Use a neutral grey sheet, keep their pick, don't touch hearts or streak. SHIPPED IN PLAYGROUND — try Tweaks → Server sim → flaky
TF-04UXNo async state on CheckWhile the server grades, the button stays static — kids double-tap. Spinner + "Checking…" label. SHIPPED IN PLAYGROUND

4 · Matching matching-v2.tsx

The thread-drag with the dashed rubber-band line is the best moment in the whole family — keep it. The gaps are around the edges of that interaction.

IDTypeFinding → recommendation
MT-01A11YKeyboard users cannot match at allTiles are buttons but all logic hangs off pointerdown — Enter/Space do nothing. Mirror the click-click semantics on keydown: Enter picks a tile, Enter on the other column pairs it. SHIPPED IN PLAYGROUND
MT-02ADAPTFixed 80px column gap starves narrow panesAt 320px each column gets ≈106px; long words overflow. Scale the gap with the container (clamp(28px, 12cqw, 80px)) and let tile text wrap. SHIPPED IN PLAYGROUND
MT-03UXAnswer reveal is one unreadable line8 pairs joined as "a → b, c → d, …" in the sheet. Render a structured two-column list, scrollable past 4 rows. SHIPPED IN PLAYGROUND
MT-04UXAttempt pool silently changes meaningDefault maxAttempts = pairs.length: 4 hearts on one task, 8 on the next. With the icon-row hearts (SH-05) this becomes visible noise. Cap at 3 like the rest of the family. SHIPPED IN PLAYGROUND
MT-05ADAPTMany pairs + phone = drag across the foldtouch-action:none tiles block page scroll mid-column; with 8 pairs at 320px the lower tiles sit below the fold. Click-click (already supported) is the touch fallback — the hint line should say so on coarse pointers; consider chunking >6 pairs into pages. PROPOSED

5 · Ordering ordering-v2.tsx

IDTypeFinding → recommendation
OR-01BUGDrag math assumes every row is 56pxSLOT = 56 + gap is a constant, but .fb-dragrow clips long content (fixed height:56px) — and any wrapped row breaks slot targeting: the ghost gap opens at the wrong index. Measure real row heights and compute targets cumulatively. Try the "Long sentences" scenario — rows wrap to 2–3 lines and drag still lands true. SHIPPED IN PLAYGROUND
OR-02A11YDrag-only reorderingNo keyboard path. Focus a row → ↑/↓ moves it one slot, with visible arrow buttons appearing on focus and an aria-live "moved to position 3 of 5". SHIPPED IN PLAYGROUND
OR-03UXRetry gives no sense of progress"Order kept" is good, but the child doesn't know which rows are right. When grading locally, say "2 of 5 already in the right place" and glow those rows green during the retry. Partial correctness becomes visible, stress drops. SHIPPED IN PLAYGROUND
OR-04ADAPTRows block page scroll on touchSame touch-action:none trade-off as matching. Acceptable while lists fit one screen; for >7 items prefer the ↑/↓ buttons on touch or a dedicated drag handle (grip-only drag). PROPOSED

6 · Fill blanks fill-blanks-v2.tsx

IDTypeFinding → recommendation
FB-01UXEmpty blanks are nearly invisible.fb-slot has color:transparent and a bottom border only; the only content is a single "·". Seven-year-olds hunt for where words go. Give empty slots a soft well background, readable "· · ·" placeholder, and a "here!" label when armed. SHIPPED IN PLAYGROUND
FB-02A11YPlacements are silent for screen readersThe fly animation is purely visual. Announce "Placed 'roots' in blank 2" via a visually-hidden live region; label slots with their state. SHIPPED IN PLAYGROUND
FB-03UXNetwork failure clears nothing — but reads as failureSame as TF-03: neutral sheet, words stay placed, retry re-submits. SHIPPED IN PLAYGROUND
FB-04UXDouble opacity on used pillsUsed pills get .locked (0.55) and inline opacity:0.25 ≈ 0.14 effective — they vanish. One opacity, one source of truth. SHIPPED IN PLAYGROUND
FB-05ADAPTSentence type doesn't scale down19px / line-height 2.3 with inline 92px slots gets raggedy at 320px. Scale with the container: clamp(16px, 4cqw, 19px), slot min-width 72px under 560px. SHIPPED IN PLAYGROUND

7 · Categorize categorize-v2.tsx

IDTypeFinding → recommendation
CT-01A11YChips are divs — keyboard and assistive tech locked outNo focus, no Enter, drag is the only path. Make chips buttons and add tap-to-arm → tap-a-bucket: it's simultaneously the keyboard path, the screen-reader path, and a friendlier touch path than long drags. Armed chips pulse the buckets as targets. SHIPPED IN PLAYGROUND
CT-02UXInstant mode interrupts with a full sheet on every wrong dropWrong drop → bottom sheet → dismiss → resume. For a sorting game the rhythm matters: shake the bucket, pulse a heart away, let the child keep sorting. Bring the sheet only at 0 hearts. SHIPPED IN PLAYGROUND
CT-03UXBucket labels are 10px grey mono uppercaseThe single most important text on screen — where things belong — is the least legible. 14px sans bold + a colour swatch per bucket gives pre-readers a second channel. SHIPPED IN PLAYGROUND
CT-04UXReveal-on-failure is a run-on stringSame as MT-03 — structured "item → group" list. SHIPPED IN PLAYGROUND
CT-05ADAPTBucket row doesn't gridflex:1 buckets wrap unevenly with 4 categories at 768px (3+1 orphan). grid-template-columns:repeat(auto-fit,minmax(150px,1fr)) keeps buckets balanced at every width. SHIPPED IN PLAYGROUND

8 · Cross-cutting & adaptiveness

The container-query move is the big one. Exercises never own the viewport — they live in lesson panes, previews, split views. Making .lf-shell a query container means every rule written once works in all of them, including the playground's device stage. This is a ~20-line diff in globals.css with no markup changes.
IDTypeFinding → recommendation
XC-01UXPartial correctness is under-celebratedMatching locks correct pairs, fill-blanks keeps correct words — good! But the sheet copy rarely says how much is right. Every retry sheet should lead with the win: "3 of 5 matched!". Implemented in ordering/fill-blanks/categorize sheets. SHIPPED IN PLAYGROUND
XC-02UXError tone: coral everywhereCoral ✕ is correct for "wrong answer" but wrong for "the system failed" (network, empty config). Three tones, consistently: green = success, coral = miss, grey = it's-not-you. SHIPPED IN PLAYGROUND
XC-03A11YReduced motion: covered, with one gapThe global prefers-reduced-motion clamp works. But flyClone/flyXP set inline transitions that bypass it. Guard the helpers with a media check (instant commit, no clone). PROPOSED
XC-04UXPlayfulness should be a dial, not a constantThe CSS already parameterises every animation via --mamp/--mdur — expose it. Younger groups get bouncier (1.45×), test-mode gets calm (0.5×). The playground Tweaks panel drives exactly these two variables. SHIPPED IN PLAYGROUND
XC-05UXStreak pill is invisible at its most excitingCoral-on-coral at streak 0 and streak 7 look identical. Light the pill (sun tones) once a streak is alive. SHIPPED IN PLAYGROUND
