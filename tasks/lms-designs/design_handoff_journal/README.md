# Handoff: GrassLMS — модуль «Журнал» (админ/координатор/методист/преподаватель)

## Обзор
Набор экранов для административной части LMS, закрывающий 6 болей:
1. составлять расписание заранее на много групп;
2. ловить пересечения кабинетов;
3. быстро проставлять посещаемость (мало кликов);
4. видеть движение группы по программе (отставание/опережение);
5. видеть, что студент реально сделал на уроке;
6. ясная структура вместо «всё на одной странице».

Ключевая идея архитектуры — **разделение трёх слоёв данных**: план (расписание,
программа) · экземпляры (занятия на даты) · факт (посещаемость, активность).
Каждый экран работает со своим слоем. Подробности — в `DATA_MODEL.md`.

## О файлах в этом пакете
Файлы здесь — **дизайн-референсы, сделанные в HTML/React (Babel-in-browser)**.
Это прототипы, показывающие внешний вид и поведение, **а не продакшн-код для
копирования**. Задача — **воспроизвести эти экраны в вашем кодовом окружении**
(у вас это React + Tailwind 4, судя по `tokens.css` и `globals.css`), используя
существующие компоненты, паттерны и API. Логику отметки/расчётов брать из
`DATA_MODEL.md`, а не из демо-данных в `admin-shell.jsx`.

## Уровень проработки
**High-fidelity.** Финальные цвета, типографика, отступы, состояния и
интерактив. Воспроизводить пиксель-в-пиксель на токенах GrassLMS (`tokens.css`).
Демо-данные (`window.*` в `admin-shell.jsx`) — мок; заменить на реальные запросы.

## Как смотреть прототип
- `prototype.html` — **сквозной кликабельный прототип** (главный артефакт):
  сайдбар переключает страницы; на «Сегодня» клик по занятию открывает деталь
  (отметка + активность); клик по ученику открывает «что он сделал за день».
- `admin-canvas.html` — **разведка вариантов** на дизайн-канвасе (по 1–2 варианта
  на экран; журнал и программа сведены к выбранным). Для сравнения, не для шипа.

Открыть в браузере (нужен интернет — React/Babel с CDN).

## Технологический контекст и токены
- **Шрифты:** `Manrope` (sans, 400–800), `Geist Mono` (mono, 400–600).
- **Токены:** `tokens.css` — это drop-in для `@theme` в `globals.css`. Использовать
  семантические переменные (`--color-primary`, `--ink-*`, `--green-*`, `--coral-*`,
  `--sun-*`, `--radius-*`, `--s-*`, `--shadow-*`). Не вводить новые цвета.
- **Иконки:** инлайновые SVG в стиле Lucide (stroke-2) — см. `shell.jsx` (`Icon`)
  и `cabinet-icons.jsx`. В вашем коде используйте `lucide-react` с теми же именами.
- **Подпись кнопки действия:** signature shadow-pop (`box-shadow: 0 3px 0 0 var(--green-700)`,
  при :active уезжает вниз). См. `.gp-btn` в исходном `index.html`.

### Палитра статусов посещаемости (используется везде)
| Статус | Код | Заливка ячейки | Текст | Точка |
|---|---|---|---|---|
| Присутствовал | `present` «П» | `--green-100` | `--green-800` | `--green-600` |
| Опоздал | `late` «О» | `--sun-100` | `--sun-700` | `--sun-500` |
| Отсутствовал | `absent` «Н» | `--coral-50` | `--coral-700` | `--coral-500` |
| Уважительная | `excused` «У» | `--ink-100` | `--ink-500` | `--ink-300` |

### Палитра статусов pacing
| Статус | Бейдж bg / fg | Полоса |
|---|---|---|
| Идёт ровно `ontrack` | `--green-50` / `--green-800` | `--green-600` |

| Опережает `ahead` | `--sun-50` / `--sun-700` | `--sun-500` |

## Общая оболочка (AdminShell)
Файл: `admin-shell.jsx`.
- **Сайдбар** 200px, фон `--ink-900`. Логотип GrassLMS (зелёный квадрат `g` с
  жёлтой точкой). Раздел «Журнал», пункты: Сегодня · Журнал · Программа ·
  Расписание · Кабинеты. Активный пункт — фон `--green-600`, белый текст.
  Внизу — карточка пользователя + роль.
- **Main** — `flex:1`, скролл по Y. Внутри — `PageHead` (заголовок 23px/800 +
  подзаголовок 13px `--ink-500` + слот `right` под фильтры/кнопки).
- Роли (admin/координатор/методист/преподаватель) меняют **scope данных и
  фильтры**, а не вёрстку. Один набор экранов.

## Экраны

### 1. Сегодня (агенда дня) — `TodayAgenda` (app-pages.jsx)
- **Назначение:** точка входа; все занятия дня по всем группам, по времени.
- **Layout:** PageHead (справа — степпер дат ‹ Сегодня › + фильтр преподавателя);
  строка из 3 мини-метрик (Занятий сегодня / Отмечено / Ждут отметки) — карточки
  flex:1; затем карточка-список занятий.
- **Строка занятия** (grid `84px · minmax(150,1.3fr) · 116px · minmax(120,1fr) · 132px`,
  gap 14, padding 15×20, hover bg `--ink-50`, cursor pointer):
  время (mono 12.5/700) · группа (цветная полоса 8×36 + название 14/800 +
  преподаватель 11.5 `--ink-400`) · кабинет (иконка `Pin`/`Video` + имя, онлайн →
  `--info`) · тема · статус посещаемости + `ChevronRight`.
- **Статус:** проведено и отмечено → зелёная точка + `present/total`; иначе →
  пустой кружок (border `--ink-300`) + «не отмечено».
- **Клик по строке** → открывает `SessionDetail` (slide-over).

### 2. Деталь занятия — `SessionDetail` (slide-over, app-pages.jsx)
- **Слой:** фикс. правая панель 470px + затемнение `rgba(10,26,16,.35)`.
- **Шапка:** название группы 18/800, время+кабинет, крестик закрытия.
- **Тело (скролл):**
  - тумблер «Занятие проведено» (38×22 pill, on → `--green-600`);
  - поле «Тема урока» (input, border 1.5px `--ink-100`);
  - **Посещаемость:** заголовок + счётчик `present/total` + кнопка «✓ Все
    присутствуют»; список ростера — имя + сегменты П/О/Н/У (30×28, выбранный
    залит цветом статуса, остальные — `inset 0 0 0 1.5px --ink-100`). Клик по
    сегменту = выставить статус (мгновенно).
  - **Активность на уроке** (если проведено): строки-кнопки студентов
    (аватар-инициал + имя + «N упр · M%» mono + Chevron) → клик открывает
    `StudentActivity`. Если не проведено — заглушка.
- **Футер:** Отмена / Сохранить (зелёная shadow-pop).
- **API при сохранении:** один батч-UPSERT `PUT /sessions/:id/attendance`
  с массивом `[{student_id, status}]` (см. DATA_MODEL §4) + `PATCH /sessions/:id`
  (held, topic/actual_topic_id).

### 3. Активность ученика за день — `StudentActivity` (full page, app-pages.jsx)
- **Назначение:** «что студент реально сделал за сегодня».
- **Layout:** кнопка «← Назад»; шапка (аватар 56 градиент green, имя 24/800,
  группа + дата); **KPI-полоса из 5 карточек** (Уроков посещено / Упражнений /
  Верных ответов % / Время в системе / Заработано XP — иконка в плашке + число +
  eyebrow); затем 2 колонки `minmax(0,1fr) · 320px`.
- **Левая колонка — уроки дня:** карточка на урок: header (полоса цвета группы +
  время mono + группа/тема + бейдж «N упр / M% верно» или «Отсутствовал»). Тело —
  список упражнений: цветная точка результата + название + тип (mono) + счёт
  `items` + бейдж результата.
  - **Результаты упражнений:** `done`(теория, серый) · `correct`(зелёный) ·
    `partial`(sun) · `wrong`(coral) · `skipped`(серый). См. `RES` в admin-shell.
  - Если отсутствовал — тело-заглушка.
- **Правая колонка — «Лента событий»:** вертикальный таймлайн (линия `--ink-100`,
  точки цвета результата): время mono + текст события (вошёл в урок, выполнил
  упражнение …).
- **Источник данных:** агрегат `activity` из попыток упражнений LMS, привязанных
  к занятию по (student_id, date). См. DATA_MODEL §1.5.

### 4. Журнал — матрица посещаемости — `JournalMatrixA` (journal-matrix.jsx)
- **Layout:** PageHead (фильтры группа/период + кнопка «CSV»); grid `1fr · 236px`:
  слева матрица, справа панель риска.
- **Матрица** (Card, overflow auto, sticky шапка + sticky первый столбец +
  sticky столбец «Посещ.»):
  - строки = студенты, столбцы = даты (число + день недели mono);
  - ячейка = кнопка 30px, заливка/буква по статусу; **клик циклит** П→О→Н→У
    (тот же UPSERT, что в детали — редактирование прямо в матрице, не только в
    деталях);
  - правый столбец = % явки (мини-полоса + число, цвет по порогам ≥85/≥70/<70);
  - `tfoot` = «пришло из N» по каждой дате.
- **Панель риска:** карточка «Средняя явка» (крупное число) + карточка «В зоне
  риска» (< 85%): список студентов с числом пропусков/опозданий и % (топ-6 по
  возрастанию).
- **Чтение:** `attendance` за период (students × dates). Цвет = status.

### 5. Программа — движение группы — `PacingFlow` (pacing.jsx)
Поток из двух представлений с внутренним state (борд ⇄ лента):
- **Борд (PacingBoard)** — обзор по всем группам:
  - KPI: сколько групп идёт ровно / отстаёт / опережает;
  - таблица групп: цвет+название+препод/кабинет · полоса прогресса `covered/total`
    с **чёрной меткой «план на сегодня»** (заливка левее метки = отставание,
    правее = опережение) · следующая тема · бейдж статуса · Chevron;
  - **клик по строке → лента этой группы**.
- **Лента (PacingTimeline)** — scope & sequence одного курса:
  - кнопка «← Все группы»; 4 KPI (пройдено тем / текущая тема / занятий /
    темп);
  - **горизонтальная лента тем**: сегмент на тему, ширина ∝ `planned_lessons`;
    цвет: пройдено `--green-500` / сейчас `--sun-400` / следующая (dashed
    `--green-300`) / впереди `--ink-50`; под сегментом — дата прохождения;
  - маркер «сегодня» по доле пройденных занятий;
  - нижняя плашка-вывод про дрейф план/факт (зелёная/coral/sun по статусу).
- **Расчёт pacing** — DATA_MODEL §3 (позиция группы, delta по занятиям или по
  `target_date`, бейдж).

### 6. Расписание — недельный планировщик — `SchedulerWeek` (scheduler.jsx)
- **Layout:** PageHead (степпер недели + фильтр + кнопка «+ Занятие»); под ним —
  **баннер пересечений** (если есть) + **легенда «цвет → группа»**; затем сетка.
- **Сетка** (grid `46px + 6 дней`): шапка дней (Пн–Сб + дата), слева — часовая
  шкала (9–18, `ROW_H=52`). В колонке дня занятия позиционируются абсолютно
  (top по времени, height по длительности). Перекрывающиеся по времени блоки в
  одном дне раскладываются по дорожкам side-by-side (greedy lane).
- **Блок занятия:** залит **цветом группы** (`color-mix(in srgb, color 16%, paper)`),
  левая граница 4px цвета группы; название группы 11/800 + кабинет (Pin/Video).
- **Пересечение** (одна комната + перекрытие времени): блок красный
  (`--coral-50` + outline `--coral-500`) + иконка `AlertTriangle`. Алгоритм —
  `computeClashes()` (DATA_MODEL §2.1, правило `a.start < b.end && b.start < a.end`).
- **API:** сохранение слота → проверка конфликта → `409` + список; кнопка
  «Сохранить всё равно» = `?force=true`.

### 7. Кабинеты — `RoomBoard` (scheduler.jsx)
- **Layout:** PageHead (бейдж «пересечение» + фильтр филиалов); таблица
  кабинеты × часы за день.
- **Строка-кабинет:** цвет + имя (+ иконка Video для онлайн) + филиал/вместимость.
- **Ячейка:** занято → зелёная плашка с названием группы (в первом часе);
  пересечение → красная плашка + `AlertTriangle`. Правый столбец — % загрузки.
- Онлайн-комнаты (Zoom/Jitsi) тоже бронируются и проверяются на пересечение —
  у них `kind='online'`, `meeting_url` (как комнаты в Google Calendar).

## Интерактив и поведение (сводка)
- **Навигация:** сайдбар → `onNav(key)` меняет страницу (`app-root.jsx`).
- **Сегодня → деталь:** клик по строке → slide-over поверх страницы.
- **Деталь → ученик:** клик по студенту в активности → full page (поверх),
  «Назад» возвращает к детали (slide-over сохраняется под низом).
- **Отметка:** клики по сегментам/ячейкам — мгновенно (оптимистично), статус
  циклится в матрице. «Все присутствуют» — массовая простановка.
- **Анимации:** тумблер 150ms, hover-фон строк 120ms. Без избыточной анимации.

## Управление состоянием (для прототипа → заменить на реальное)
- `app-root.jsx`: `page` (today/journal/pacing/schedule/rooms), `session` (id
  для slide-over), `student` (имя для full page).
- `SessionDetail`: локальный `held`, `att[]` (статусы ростера).
- `JournalMatrixA`/`useGrid`: `grid[][]` статусов.
- `PacingFlow`: `sel` (выбранная группа).
В проде: server state через ваш data-layer (React Query/SWR), оптимистичные
мутации для отметки.

## Порядок внедрения (рекомендация)
1. **Бэкенд-основа:** `rooms` (offline+online), `schedule_slots` + `sessions`
   (генерация), детект конфликтов (DATA_MODEL §2), `attendance` + батч-UPSERT.
2. **Экраны факта:** Сегодня → деталь (отметка) → матрица. Самый быстрый эффект.
3. **Программа:** `curriculum_topics`, связь `session.actual_topic_id`, pacing.
4. **Активность ученика:** свёртка `activity` из попыток упражнений.
5. **Расписание/кабинеты:** планировщик недели + сетка кабинетов.
